1. Why is object-oriented development so important in programming?

    2. Object-oriented programming helps organize code into reusable, modular, and scalable structures. It enables developers to model real-world concepts, ensures code is easier to maintain, and reduces complexity through encapsulation.

  2. In Task 1, we create a static object called "AccountType." Explain what this object is used for and how this type of object can help us in software development.

    3. The AccountType object holds predefined account types. It standardizes classifications, enhances readability, and centralizes related constants, reducing errors and simplifying updates.

  3. Discuss why we use encapsulation in classes. Why do we want to "hide" code within our class? Explain how we can create "private" and "public" members in a constructor function.

    4. Encapsulation hides internal details of a class, improving security, usability, and maintainability. Private members use the # prefix, while public members are accessible directly. Example:
    class Example { #privateVariable = "hidden"; publicVariable = "visible"; publicMethod() { return this.#privateVariable; }}

  4. Explain what happens when we use the "new" command in JavaScript.

    5. The new command creates an instance of a class or constructor function. It binds this to the new object, sets the prototype, and returns the new object unless explicitly overridden.
