  2. In part 2, can we create what is known as an "infinite" loop within the condition of the "while" loop?

    3. Yes, an infinite loop can be created by ensuring the condition in the "while" loop always evaluates to true. For instance: while (true) { /* Code runs indefinitely */ }
    However, infinite loops must include a condition or "break" statement to prevent the program from crashing. Example: while (true) { if (someCondition) break; }

  4. Explain the relationship between rows and columns in nested "for" loops. Can we, for instance, change the order, considering rows to be innermost or columns to be innermost, or does this not matter?

    5. The order of nested loops determines how rows and columns are processed:
    - **Rows Outer, Columns Inner**: Processes one row at a time, completing all columns before moving to the next row.
    - **Columns Outer, Rows Inner**: Processes one column at a time, completing all rows before moving to the next column.
    While the order of processing differs, the result is equivalent if data is organized accordingly .
