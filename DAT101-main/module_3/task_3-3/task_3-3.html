1. What is the difference between invoking and creating a function?

    2. Creating a function involves defining it using function or arrow syntax. Invoking a function runs it by calling its name followed by parentheses.
    Example: function greet() { console.log("Hello!"); } greet();

  2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons terminate statements but are optional due to automatic semicolon insertion. Code blocks use curly braces {} to group statements.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. The variable will be undefined. Example: function noReturn() {} let result = noReturn(); // result is undefined.

  4. How do we separate parameters from each other when creating a function?

    5. Parameters are separated by commas in the function declaration and when passing arguments during invocation.
    Example: function example(a, b, c) {}

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. Parameters act as placeholders. During invocation, arguments map to these placeholders in the same order.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. A missing parameter defaults to undefined unless a default value is provided.

  7. Explain what happens when we use recursive functions.

    8. Recursive functions call themselves and require a base case to avoid infinite recursion. Example: function factorial(n) { if (n === 0) return 1; return n * factorial(n - 1); }
